# 15. 3Sum

[:paperclip: LeetCode Problem Description](https://leetcode.com/problems/3sum/description/)

## Solution: Sorting and Two Pointers

The goal is to find all unique triplets in the array that sum up to zero. By sorting the array, we can use a two-pointer approach for each element to find the other two numbers, similar to the "Two Sum II" problem.

### Thought Process

1.  **Sorting**: Sort the input array `nums`. This allows us to easily skip duplicate values and use two pointers to converge on the target sum.
2.  **Outer Iteration**:
    - Iterate through the array with index `i`.
    - **Optimization (Pruning)**: Since the array is sorted, if `nums[i] > 0`, it is impossible to find two more numbers to its right that sum to zero (as they would all be positive).
    - **Duplicate Handling**: If `nums[i]` is the same as the previous element `nums[i-1]`, skip it to avoid duplicate triplets.
3.  **Two Pointers**:
    - For each `nums[i]`, set two pointers: `l = i + 1` and `r = n - 1`.
    - Calculate `sum = nums[i] + nums[l] + nums[r]`.
    - **Sum == 0**: We've found a triplet. Add it to the result.
        - Increment `l` and decrement `r`.
        - **Skip Duplicates**: Move `l` forward as long as `nums[l] == nums[l-1]` to ensure we don't pick the same number again.
    - **Sum > 0**: The sum is too large. Decrement `r` to move toward smaller values.
    - **Sum < 0**: The sum is too small. Increment `l` to move toward larger values.

### Go Code

``` go
import "sort"

func threeSum(nums []int) [][]int {
    sort.Ints(nums)
    n := len(nums)
    res := make([][]int, 0)
    for i := 0; i < n-2 && nums[i] <= 0; i++ {
        if i > 0 && nums[i-1] == nums[i] {
            continue
        }
        l, r := i+1, n-1
        for l < r {
            sum := nums[i] + nums[l] + nums[r]
            if sum == 0 {
                res = append(res, []int{ nums[i], nums[l], nums[r] })
                l++
                r--
                for l < r && nums[l-1] == nums[l] {
                    l++
                }
            } else if sum > 0 {
                r--
            } else {
                l++
            }
        }
    }
    return res
}
```

### Code Efficiency

- **Time Complexity**: $O(n^2)$
    - Sorting the array takes $O(n \log n)$.
    - The nested loops (one outer and the two-pointer inner) take $O(n^2)$.
- **Space Complexity**: $O(\log n)$ or $O(n)$
    - Depending on the sorting implementation, it requires some auxiliary space. Go's `sort.Ints` typically uses $O(\log n)$ auxiliary space.
